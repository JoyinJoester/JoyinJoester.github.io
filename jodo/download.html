Linux 环境要求
  • Linux 内核 4.0 或更高版本
  • Rust 1.55.0 或更高版本
  • Cargo 包管理器
  • Git
  • gcc 或兼容的C编译器
Linux 构建步骤

克隆仓库

git clone https://github.com/JoyinJoester/JODO.git cd JODO

构建项目

cargo build --release

安装到系统

sudo cp target/release/jodo /usr/local/bin/

运行测试

cargo test
在 Linux 上使用 Cargo 安装

如果您已经安装了 Rust 工具链，可以直接使用 Cargo 安装 JODO：

cargo install --git https://github.com/JoyinJoester/JODO.git

这将自动下载、编译和安装 JODO 到您的系统中。

Cargo 安装优势
  • 自动处理依赖关系
  • 简化安装过程
  • 自动添加到您的系统路径
  • 方便后续更新
更新 JODO
cargo install --git https://github.com/JoyinJoester/JODO.git --force
在 Linux 上验证安装

检查版本

jodo --version

如果显示版本信息，则表示安装成功！

查看帮助信息

jodo --help

显示所有可用的命令和选项。

检查配置路径

jodo --config-path

查看 JODO 存储配置文件的位置。

Linux 上开始使用

创建第一个任务

jodo "我的第一个待办事项"

添加带截止日期的任务

jodo "重要任务" -t 2023-12-31

查看任务列表

jodo -l

快捷命令
命令 功能
jodo -l 列出所有任务
jodo -c 1 完成ID为1的任务
jodo -d 1 删除ID为1的任务
jodo -p 只显示优先级任务
Linux 常见安装问题

权限错误

如果在 Linux 上安装时遇到权限问题，请使用：

sudo chmod +x /usr/local/bin/jodo

找不到命令

确保 /usr/local/bin/ 在系统 PATH 环境变量中。如果不确定，可以通过编辑 ~/.bashrc 文件添加：

export PATH=$PATH:/usr/local/bin

依赖问题

如果从源码构建时遇到依赖错误，请确保您的 Rust 环境是最新版本：

rustup update

安装缓慢

从源码构建可能需要一些时间，特别是在第一次编译依赖项时。请耐心等待。
